# SAQ Digital Solutions — Microservices Platform

Платформа веб-присутствия креативного агентства **SAQ Creative Agency** (Астана). 

Проект переведен с монолитной архитектуры на современную архитектуру микросервисов, оркестрируемую через **Docker Compose**. Все запросы проходят через единую точку входа (API Gateway) с лимитированием частоты запросов, а данные хранятся в общей базе данных **PostgreSQL 16**.

---

## Стек технологий

*   **API Gateway:** Nginx
*   **Бэкенд микросервисы:** Node.js (Express), Multer, Express Rate Limit, pg (node-postgres)
*   **Фронтенд:** Ванильный HTML/CSS/JS, GSAP + ScrollTrigger (анимации), шрифты Outfit + Geist Mono
*   **База данных:** PostgreSQL 16 (официальный образ `postgres:16-alpine`)
*   **Оркестрация:** Docker / Docker Compose

## Развертывание и запуск

Для запуска всей платформы в докере необходим установленный Docker и Docker Compose.

### 1. Настройка переменных окружения
Создайте файл `.env` в корневой директории на основе следующих параметров:

```ini
# Пароль от панели управления админа
ADMIN_PASSWORD=your_secure_password

# Настройки уведомлений Telegram (опционально)
TELEGRAM_BOT_TOKEN=123456:ABC-DEF...
TELEGRAM_CHAT_ID=-100200300400
TELEGRAM_TOPIC_ID=123  # Если используется группа с темами (topics)

# Настройки уведомлений SMTP (опционально)
SMTP_HOST=smtp.mailru.ch
SMTP_PORT=465
SMTP_USER=leads@yourdomain.com
SMTP_PASS=your_smtp_password
LEAD_EMAIL_TO=admin@yourdomain.com
```

### 2. Запуск одной командой
В корневой папке проекта выполните команду:

```bash
# Запуск и сборка контейнеров в фоновом режиме
npm run docker:up
```
*Или напрямую через docker compose:* `docker compose up -d --build`

После этого API Gateway будет доступен по адресу **`http://localhost:3000`**.

### 3. Полезные скрипты
В корневом `package.json` настроены вспомогательные команды:
*   `npm run docker:up` — сборка и запуск системы в Docker.
*   `npm run docker:down` — остановка и удаление контейнеров.
*   `npm run docker:logs` — просмотр логов всех сервисов в реальном времени.

## Пагинация данных (Pagination)

Для оптимизации трафика и удобства интерфейса внедрена пагинация:
1.  **Фронтенд (Главная страница)**: блок портфолио отображает **по 4 кейса на страницу** с сохранением плавных анимаций переходов (GSAP ScrollTrigger).
2.  **Админ-панель (Заявки)**: таблица заявок отображает **по 10 записей на страницу**.
3.  **Админ-панель (Портфолио)**: список кейсов для редактирования отображает **по 5 записей на страницу**.

## Безопасность и защита данных

1.  **Изоляция портов**: Ни один сервис (включая СУБД PostgreSQL) не выставляет свои порты наружу. Публично доступен только порт `3000` шлюза Nginx.
2.  **Защита от брутфорса**: Nginx лимитирует запросы к админке (`/admin`) до **3 запросов в секунду** с одного IP.
3.  **Смена сессий раз в час**: Авторизация в панели управления реализована через кастомный HTML-интерфейс (Basic Auth отключен на уровне всплывающих окон браузера). Введенные данные и метка времени хранятся в `sessionStorage`. **Каждый час сессия сгорает**, требуя от администратора повторного ввода пароля.
4.  **Антиспам (Honeypot)**: Форма заявок содержит невидимое для людей поле `company`. При автоматическом заполнении этого поля ботом запрос молча отбрасывается без отправки спама в Telegram/Email.
5.  **SQL-Injection Protection**: Все запросы к PostgreSQL параметризованы, прямой SQL-инъекции не подвержены.
